# 调度优化新思路

## 核心思路

### 约束条件
- 最大时间限制：3600 秒

### 奖励函数
- 奖励 = 单位时间内的产能
- 产能 = 完成工件数 / 总时间

### 形式化
```
Reward = n_wafer / total_time
```

## 优势

1. **直接优化目标** - 最大化生产效率
2. **简单直观** - 避免复杂的奖励设计
3. **公平比较** - 在相同时间约束下比较不同调度方案

## 潜在问题

1. **early termination** - 如果提前完成，奖励会很高（需要处理）
2. **探索激励** - 可能导致模型只关注快速路径

## 思考

- 是否需要归一化？
- 如何处理提前完成的情况？
- 是否需要加入惩罚项（死锁、报废等）？
